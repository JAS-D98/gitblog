Everyday Git

What is Git?

Git is a free and open source distributed version control system (VCS), authored by Linus Torvalds and officially released on 7th April 2005. Git helps to track changes in any set of computer files and is often used for coordinating work among programmers who are collaboratively developing source code during software development.

Git is used to store code, track revision history, merge code changes as well as revert to earlier code versions whenever needed. Though git is designed to handle everything from small to very large projects with speed and efficiency, it is easy to learn

Version Control (VC)

Version control is also known as source control and it is the practice of tracking and managing changes to software code.

Version Control Sytstems (VCS)

Version control systems are software tools that help software teams manage changes to source code over time thus enabling them work faster and smarter.

They keep track of every modification to the code in a special kind of database. If a mistake is made, developers can turn back the clock and compare earlier versions of the code to help fix the mistake while minimizing disruption to all team members.

GITHUB

Github is a developer platform that allows developers to create, store, manage and share their code. It uses Git software, providing the distributed version control of Git plus access control, bug tracking, software feature requests, task management, continuous integration, and wikis for every project.

  3. Initializing a Git repository

    A Git Repository is the .git/ folder inside a project. This repository tracks all changes made to files in your project, building a history over time. Meaning, if you delete the .git/ folder, then you delete your project's history.

    To initialize a git repository, we follow through the following steps;

    • Open the terminal or command prompt and navigate to the expected folder by running cd "folder name" in the terminal. Here I will be navigating to a folder named "gittutorialblog" that's based in the desktop.

      We therefore initialize the repository by running git init in the terminal

  4. Adding files to the staging area

    A staging area is a file, generally contained in your Git directory, that stores information about what will go into your next commit. Its technical name in Git parlance is the “index”, but the phrase “staging area” works just as well. This helps to track the files.

    To add files to staging area we first run git status to check whether the repository was initialized and then run git add . to add all files to staging area.
    If you just wish to add a specific file and not all the files to the staging area run git add "file name".

    Tracked files are files that Git knows about. Untracked files are all files in your working directory that were not in your last snapshot and are not in your staging area.

    They are the ones handled (version controlled) by Git, that were once added and committed. Untracked files are most of the time files you don't want to be controlled, because for example they are generated by your compiler. Untracked files are added to the .gitignore file, so that Git doesn't track them.

When you want to add a new feature or fix a bug—no matter how big or how small, you spawn a new branch to encapsulate your changes.
